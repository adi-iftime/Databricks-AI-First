# Skill: Frontend

Reusable capability definition. **Not** tied to any single agent; routing is defined in orchestration rules.

## Technologies

- Component-based UI frameworks (e.g. React) as used by the project
- TypeScript or JavaScript per repository standards
- Styling approach already present (CSS modules, design system, etc.)

## Patterns

- Controlled vs uncontrolled inputs, derived state minimization
- Accessible markup and keyboard flows where UI is user-facing
- Client-side routing and data fetching patterns consistent with the app

## Domain knowledge

- Loading/error/empty states for async data
- Basic performance hygiene (memoization only when justified)
- Cross-browser and responsive layout considerations

## Best practices

- Reuse existing components and tokens before introducing new patterns
- Keep components focused; colocate tests when the repo does so
- Prefer clarity over cleverness in UI code
